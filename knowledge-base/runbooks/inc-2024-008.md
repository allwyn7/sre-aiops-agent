Verify that DNS queries are blocked with the following commands:,   - Use `dig` from a pod in `bookshop-prod`: `kubectl exec -n bookshop-prod <pod-name> -- dig inventory-srv.inventory-prod.svc.cluster.local`.,   - Check for `NXDOMAIN` or timeouts in the response.,5. Apply the fixed policy using `kubectl apply -f bookshop-egress-policy.yaml`.,3. Redeploy affected pods in `bookshop-prod` using `kubectl rollout restart deploy bookshop-srv`.,4. Monitor DNS resolution to validate the fix using Prometheus metrics (e.g., `dns_resolution_failure_rate_percent`, `p99_response_time_ms`).
