# Runbook: JPA Entity Schema Drift Recovery

> **Incident:** INC-2024-002 | **Service:** `bookshop-srv` | **Date:** 2024-04-10

**Trigger Pattern:** `.*PSQLException: ERROR: column .* does not exist.*`

---

## 1. Detection

Look for HTTP 500 spikes in Prometheus metrics on affected endpoints (e.g., `http_5xx_rate_per_min`). Check corresponding logs for `SQLGrammarException` related to missing database columns.

## 2. Triage

1. Check affected endpoints in the alert payload (`GET /api/books`). 2. Review application logs for `SQLGrammarException`. 3. Verify database schema against JPA entity using Flyway migration state and manual inspection.

## 3. Diagnosis

Run `kubectl logs` to inspect service logs for errors like `PSQLException: column does not exist`. Use `kubectl exec` or database tools to check for schema discrepancies: `SELECT column_name FROM information_schema.columns WHERE table_name = 'book';` and cross-check DB schema with the JPA entity.

## 4. Resolution

1. Add Flyway migration: Create `db/migration/V2__drop_price_old_column.sql` with content `ALTER TABLE book DROP COLUMN IF EXISTS price_old;`. 2. Deploy migration to the database via Flyway. 3. Redeploy the application to ensure schema and application match. 4. Perform integration tests on affected endpoints to validate the fix.

## 5. Verification

1. Run health checks: `curl GET /actuator/health`. Ensure all components have `status: UP`. 2. Verify metrics: The `http_5xx_rate_per_min` should return to baseline levels. 3. Perform API tests on endpoints (`GET /api/books`, etc.) to ensure they return the expected responses. 4. Monitor application logs for at least 30 minutes to ensure no further `SQLGrammarException` or `PSQLException` errors.

## 6. Prevention

1. Implement automated CI/CD checks to ensure Flyway migrations are in sync with JPA schema changes. 2. Establish code review guidelines for database schema changes. 3. Set up ongoing schema monitoring tools to check for drift (e.g., Liquibase, Flyway validate). 4. Add enhanced validation in the CI pipeline to fail builds if Flyway migrations are not applied.
