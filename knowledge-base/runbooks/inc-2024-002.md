# Runbook: Resolve SQLGrammarException due to Schema Drift

> **Incident:** INC-2024-002 | **Service:** `bookshop-srv` | **Date:** 2024-04-10

**Trigger Pattern:** `org.postgresql.util.PSQLException: ERROR: column \"price_old\" does not exist`

---

## 1. Detection

- Alerts for HTTP 5xx errors.
- High error rate (e.g., 96%) on APIs querying `Book` entity.

## 2. Triage

1. Investigate recent Pull Requests involving changes to the `Book` entity.
2. Check database logs and Hibernate exceptions for schema or entity mismatch.
3. Identify the relevant Flyway version to fix the schema.

## 3. Diagnosis

1. Look for `SQLGrammarException: could not extract ResultSet` in logs.
2. Check if the column removed from the entity still exists in or is missing from the database.
3. Verify with the blame PR to identify missing Flyway migrations.

## 4. Resolution

1. Create a new Flyway SQL migration to align the database schema.
2. Apply changes by deploying to a staging environment.
3. Validate that endpoints are functioning normally.

## 5. Verification

- Verify column no longer queried in SQL.
- Validate all Book endpoints respond with correct data and no errors.
- Confirm Flyway migrations applied without issues.

## 6. Prevention

- Always accompany JPA entity changes with corresponding Flyway migration.
- Add pre-merge checks to validate JPA and schema consistency.
