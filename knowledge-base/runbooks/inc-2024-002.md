# Runbook: JPA Entity Schema Drift Recovery

> **Incident:** INC-2024-002 | **Service:** `bookshop-srv` | **Date:** 2024-04-10

**Trigger Pattern:** `org.postgresql.util.PSQLException: ERROR: column .* does not exist`

---

## 1. Detection

Monitor HTTP 500 spikes, logs for org.hibernate.exception.SQLGrammarException or org.postgresql.util.PSQLException, and health check failures (e.g., /actuator/health reporting DOWN due to DB errors).

## 2. Triage

1. Confirm the error pattern in logs (e.g., SQLGrammarException indicating missing column).,2. Check affected endpoints using Prometheus metrics (http_500_rate_per_min > 2x baseline).,3. Validate the database schema consistency using the application logs and a schema inspection query.,4. Verify the database health using /actuator/health probe for datasource status.

## 3. Diagnosis

1. Analyze recent application logs for SQLGrammarException and note the missing database column.,2. Identify recently merged PRs or configuration changes that may have caused inconsistency.,3. Query the database schema to confirm the missing column: SELECT column_name FROM information_schema.columns WHERE table_name = 'book';,4. Validate the Flyway migrations applied in the database: SELECT * FROM flyway_schema_history;

## 4. Resolution

1. Rollback the affected PR (PR #52 in this case) to restore the removed JPA entity field (priceOld).,2. Stage and apply a missing Flyway migration to drop the 'price_old' column from the database: db/migration/V2__drop_price_old_column.sql containing ALTER TABLE book DROP COLUMN IF EXISTS price_old;,3. Once the column is dropped, merge the previously rolled back PR back into production.,4. Restart the application pods to ensure the updated schema and entity mapping is used.

## 5. Verification

1. Confirm no SQLGrammarException logs in /var/log/application.log.,2. Validate all affected endpoints (e.g., GET /api/books) return 200 HTTP status codes.,3. Ensure /actuator/health probe returns a healthy status.,4. Verify Prometheus metrics http_5xx_rate_per_min stabilizes below the baseline (< 0.5 per minute).

## 6. Prevention

1. Enforce CI checks to validate JPA entity consistency against Flyway migration scripts before allowing PR merges.,2. Add a pre-merge checklist for schema changes to ensure migrations are created alongside entity updates.,3. Include schema drift validations in pre-deployment pipelines (e.g., Flyway validate command).,4. Configure alerts for schema inconsistencies during deployment.
