# Runbook: JPA Entity Schema Drift Recovery

> **Incident:** INC-2024-002 | **Service:** `bookshop-srv` | **Date:** 2024-04-10

**Trigger Pattern:** `org.postgresql.util.PSQLException: ERROR: column "\w+" does not exist`

---

## 1. Detection

Look for error logs containing SQLGrammarException or PSQLException related to a missing database column. Monitor health endpoints (/actuator/health) for DOWN status due to database validation failing. Alert on HTTP 500 spikes above thresholds (e.g., 5x baseline).

## 2. Triage

1. Review the error logs for missing database columns. 2. Check the recent changes in the codebase (e.g., PRs, migrations). 3. Verify the status of affected endpoints using metrics or by directly querying the service. 4. Notify the on-call SRE and service owner.

## 3. Diagnosis

1. Search logs for SQLGrammarException or invalid ResultSet exceptions. 2. Identify the associated column and reference the corresponding JPA entity. 3. Check migrations (Flyway scripts) to ensure they reflect recent entity modifications. 4. Verify schema consistency directly on the database using queries like `\\d table_name` or `SELECT * FROM information_schema.columns`.

## 4. Resolution

1. Rollback the impacting PR to restore service if immediate resolution is required. 2. Apply a Flyway migration (e.g., `ALTER TABLE book DROP COLUMN IF EXISTS price_old;`). 3. Redeploy the service after migration to update the schema. 4. Monitor logs and metrics to confirm the resolution.

## 5. Verification

1. Ensure /actuator/health reports UP status. 2. Verify no 500 errors in logs across affected endpoints (`/api/books`, `/api/books/{id}`, `/api/books/search`). 3. Perform a functional test of endpoints and compare responses to baseline behavior. 4. Check Prometheus metrics (e.g., `http_5xx_rate_per_min`) to ensure error rates are at baseline or lower.

## 6. Prevention

1. Integrate static code analysis or CI checks to ensure Flyway scripts match ORM changes. 2. Add automated schema validation tests in pre-prod pipelines. 3. Enhance monitoring to detect schema inconsistencies early (e.g., verify `information_schema.columns` against JPA entity models).
