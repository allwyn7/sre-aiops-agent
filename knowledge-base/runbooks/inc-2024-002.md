**Trigger Pattern:** `org.postgresql.util.PSQLException: ERROR: column .* does not exist`

## 1. Detection

Monitor logs for SQLGrammarException with message indicating a missing column. Alert on an HTTP 500 error rate exceeding baseline by 3-5x. Leverage `/actuator/health` endpoint for database health check failures.

Notify the service owner and database team if the database schema appears to be out of sync with the application code.

1. Use `kubectl logs <pod> -c bookshop-srv` to fetch logs around the incident timeframe. Look for SQLGrammarException indicating schema mismatch.,2. Login to the PostgreSQL instance and check the schema using: `\\d+ book`. Confirm the absence of the reported column (e.g., price_old).,3. ## 4. Resolution

1. Create a Flyway SQL migration file, e.g., `V2__drop_price_old_column.sql`, with the following contents: `ALTER TABLE book DROP COLUMN IF EXISTS price_old;`.,2. Test the migration in a lower environment:,   a. Deploy the updated schema migration script.,   b. Verify the database schema is in sync using Flyway commands such as `mvn flyway:info`.,   c. Test affected endpoints to ensure no errors persist.,3. Deploy the schema migration to the production environment following change management procedures.,4. Resolve affected database health alerts and monitor logs for recurring SQL errors or related abnormalities.

## 5. Verification

1. Verify `/actuator/health` endpoint returns `UP` and database health is inspected.,2. Confirm HTTP 500 error rate drops back to the baseline via monitoring metrics like `http_server_requests`.,3. Execute database query tests in production to confirm the schema and ORM consistency: `SELECT price_old FROM book` should not return an error.

## 6. Prevention

1. Ensure CI pipelines include automated database migration validation (e.g., Flyway validation during build).,2. Implement pre-merge checks to detect schema changes mismatched between JPA entities and Flyway migrations.,3. Monitor database migration history (`flyway_schema_history`) and compare against entity definitions periodically to catch discrepancies early.
