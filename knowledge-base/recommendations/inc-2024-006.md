## Disaster Recovery Assessment

### Single Points of Failure
- Single replica/low HPA maxReplicas, leading to complete failure of service.
- Insufficient node capacity due to static node pool size.

### High Availability Recommendations
Increase HPA maxReplicas, add pod anti-affinity rules, and ensure the node pool is auto-scalable with high availability zones.

### Failover Strategy
Configure multi-zone clusters and use affinity rules to spread pods across nodes. Leverage external load balancers for traffic routing.

### RTO / RPO Assessment
Ensure RTO of 15-20 mins (resolve incident or failover switch) and RPO of 5 minutes via frequent backups of stateful data.

### Backup Recommendations
Implement regular, automated database backups using SAP HANA services or custom PostgreSQL backup scripts. Test restore procedures quarterly.
