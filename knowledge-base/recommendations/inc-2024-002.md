## Performance Tuning

### JVM Tuning
-Xms1g -Xmx2g, use G1GC with flags -XX:+UseG1GC -XX:MaxGCPauseMillis=200.

### Spring Boot Configuration
Increase server.servlet.session.timeout=30m and enable lazy initialization (spring.main.lazy-initialization=true).

### JPA / Hibernate Tuning
Enable batching for JPA queries (spring.jpa.properties.hibernate.jdbc.batch_size=20) and set hibernate.query.fail_on_pagination_over_collection_fetch=true.

### Database (PostgreSQL) Tuning
Set PostgreSQL shared_buffers to 25% of total memory and work_mem to 4MB.

### Connection Pool (HikariCP) Tuning
Set maximumPoolSize to 15 and idleTimeout to 30000ms for HikariCP.

### Kubernetes Tuning

Set resource requests for the pods as limits to prevent overcommitment. Use HPA to scale on CPU ≥ 70% or memory utilization ≥ 80%. Define PodDisruptionBudget with min-available replicas >= 1.

### Network Tuning

Configure retries in HTTP clients via Spring Cloud to handle transient errors. Ensure network policies enforce secure DB access.
